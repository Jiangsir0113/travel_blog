# 旅行博客初始化设计

## 目标

初始化一个名为 `travel_blog` 的公开 GitHub 仓库，并搭建旅行博客的第一版。第一版先采用内容优先的静态站点形态，方便持续写游记，同时为后续加入更强的视觉作品集效果和旅行工具功能预留空间。

## 仓库与项目设置

- 将本地项目文件夹从 `personal_blog` 改名为 `travel_blog`。
- 创建名为 `travel_blog` 的公开 GitHub 仓库。
- 使用 `main` 作为默认分支。
- 首次提交保持清晰克制：包含项目脚手架、基础内容结构和必要文档。
- 所有文档、代码注释和 Git 提交信息均使用中文。

## 推荐技术栈

使用 Astro 作为静态站点框架。

Astro 适合第一版旅行博客，因为它天然支持 Markdown 内容，默认输出静态 HTML，运行和部署都比较轻。后续需要加入照片墙、地图、搜索或行程组件时，也可以用 Astro 的交互组件能力逐步扩展。

## 第一版范围

第一版应包含：

- 一个内容优先的首页，展示最近游记、目的地入口和少量照片预览。
- 基于 Markdown 或 MDX 的游记文章。
- 一个游记内容集合，文章 frontmatter 包含标题、日期、目的地、摘要、主图、标签，以及可选坐标。
- 基础目的地页面或路由，后续可以扩展为更完整的目的地指南。
- 存放公开图片和未来照片墙素材的位置。
- 一份 README，说明如何运行、写作和发布博客。

## 后续扩展方向

项目结构应为以下能力预留空间：

- 视觉作品集方向：照片墙、目的地专题页、沉浸式首屏和精选旅程合集。
- 旅行工具方向：地图、行程时间线、地点收藏、搜索筛选和路线视图。
- 无需后端即可部署到 GitHub Pages 或 Netlify。

## 第一版不包含

初始版本不包含登录认证、数据库、评论系统、CMS 集成、高级地图或完整行程规划器。这些功能可以在静态写作流程稳定后逐步加入。

## 测试与验证

实现后需要验证：

- 依赖可以成功安装。
- 开发服务器可以启动。
- 生产构建可以通过。
- 首页和至少一篇示例游记可以正常渲染。
- 本地 git 仓库已连接到公开 GitHub 仓库。
